# Rebrand Notes

## Rule

The scaffold is designed to be rebranded without changing the underlying behavior.

## What Must Be Replaceable

- App name and logo
- Color system and typography
- Copy, labels, and onboarding text
- Channel naming and category taxonomy
- Premium naming and upsell wording
- Empty states, illustrations, and mascot assets

## What Should Stay Stable

- Install identity model
- City-based feed structure
- Adult gate flow
- Wallet and ledger model
- Creator review and moderation workflows
- API contracts and data model shape

## Rebrand Order

1. Replace brand tokens
2. Replace copy
3. Replace visuals
4. Keep API and data contracts stable
