Hier ist der Entwurf für deine Datenbank-Struktur und die Logik-Pfade, basierend auf den Bildern:

1. Das Datenbank-Schema (ER-Diagramm)
Um diese App nachzubauen, benötigst du diese zentralen Tabellen und Verknüpfungen:

Kern-Tabellen
Users: Speichert User_ID, Karma_Punkte, Heimat_Ort, Device_Token, Is_Plus_Member (Boolean) und Join_Date.

Posts (Jodel): Post_ID, User_ID (FK), Content (Text/Bild/Video), Latitude/Longitude, Channel_ID (FK), Color_Hex, Vote_Count, Comment_Count und Expires_At (für Umfragen/Timer).

Comments: Comment_ID, Post_ID (FK), User_ID (FK), Content, Vote_Count und Parent_Comment_ID (für Antworten auf Kommentare).

Interaktions-Tabellen
Votes: User_ID, Target_ID (Post oder Comment), Vote_Type (Up/Down). Wichtig, um Doppel-Votes zu verhindern.

Channels: Channel_ID, Name, Member_Count, Is_NSFW (Boolean), Is_Verified.

Subscriptions: Verknüpft Users mit Channels (wer hat welchen Channel abonniert).

Chats: Chat_ID, Initiator_ID, Receiver_ID, Status (Anfrage/Aktiv).

2. Die Logik-Diagramme (Flows)
A. Der Feed-Algorithmus (Location-Based)
Dieser Prozess läuft ab, wenn du die App öffnest:

Standort-Check: App sendet Lat/Long an den Server.

Radius-Logik: Server berechnet Distanz: SELECT * FROM Posts WHERE distance(User_Loc, Post_Loc) < Dynamic_Radius.

Filter-Anwendung: Sortierung nach Neu, Kommentiert oder Lauteste (höchste Votes).

Channel-Filter: Wenn User in @beichtstuhl ist, zeige nur Posts mit dieser Channel_ID.

B. Das Karma & Voting System
Das ist dein "Währungssystem" in der App:

Aktion: User klickt "Upvote".

Logik: * Prüfe in Tabelle Votes, ob User bereits gevotet hat.

Wenn Nein: Update Posts SET Vote_Count = Vote_Count + 1.

Karma-Update: Update Users SET Karma = Karma + 1 (beim Ersteller des Posts).

Schwellenwert: Bestimmte Features (wie "Bilder im Chat") werden erst ab einem Karma-Wert in der Users-Tabelle freigeschaltet.

C. Chat-Anfragen Logik
Basierend auf deinen Screenshots:

Trigger: User A klickt auf "Antworten" bei einem Post von User B.

Anfrage: Eintrag in Chats mit Status = 'Pending'.

Limit-Check: System prüft Users.Daily_Chat_Requests. (Normale User haben ein Limit, Plus-User mehr).

Notification: User B erhält einen Eintrag in der Notifications-Tabelle.

3. Besonderheiten für den Nachbau
Hashtag-System: Du brauchst eine Tabelle Hashtags und eine Zwischentabelle Post_Hashtags, um die "Trending"-Sektion zu füttern.

Push-Einstellungen: In der Users-Tabelle oder einer Settings-Tabelle müssen die Toggles für jeden Notification-Typ (Votes, Replies, Mentions) gespeichert werden.

Plus-Features: Viele Funktionen sind hinter einem Is_Plus_Member-Flag gesperrt (z. B. "Explorer Modus" oder "Keine Werbung").
